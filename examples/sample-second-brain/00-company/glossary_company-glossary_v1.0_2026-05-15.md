# Glossary — SecondBrain Co.

**Phiên bản:** 1.0
**Ngày:** 2026-05-15

---

| Thuật ngữ | Định nghĩa |
|-----------|-----------|
| **Second Brain** | Hệ thống tri thức cá nhân bên ngoài bộ nhớ sinh học — nơi lưu trữ, tổ chức và khai thác thông tin có hệ thống. |
| **Knowledge Gem** | Insight hoặc piece of knowledge có giá trị cao, rare, có thể áp dụng vào nhiều ngữ cảnh khác nhau. |
| **Vault** | Kho lưu trữ tri thức trung tâm (thường là DEVONthink, Obsidian, hoặc Notion database). |
| **Capture** | Hành động thu thập thông tin từ nguồn bên ngoài vào hệ thống — là bước đầu tiên trong knowledge pipeline. |
| **Curation** | Quá trình lọc, tổ chức, gắn tag và tạo liên kết giữa các knowledge notes. |
| **Synthesis** | Kết hợp insights từ nhiều nguồn/domain khác nhau để tạo ra understanding mới (higher-order thinking). |
| **Knowledge Pipeline** | Chuỗi các bước từ capture → process → curate → apply → grow. Toàn bộ value chain của SecondBrain Co. |
| **Evergreen Note** | Note được viết để tồn tại lâu dài và luôn được cập nhật — không phải fleeting note tạm thời. |
| **Fleeting Note** | Ghi chú tạm thời, capture nhanh — cần được process thành atomic note hoặc project note sau đó. |
| **Atomic Note** | Note chỉ chứa 1 idea hoàn chỉnh — đủ độc lập để được link từ nhiều nơi khác. |
| **MOC (Map of Content)** | Note index cho 1 topic — liên kết các atomic notes liên quan, tạo overview và navigation. |
| **Progressive Summarization** | Kỹ thuật tóm tắt dần dần: bold → highlight → executive summary — chắt lọc essence qua nhiều pass. |
| **Spaced Repetition** | Kỹ thuật ôn tập theo khoảng cách thời gian tăng dần — tối ưu retention với effort tối thiểu. |
| **Zettelkasten** | Hệ thống quản lý notes của Niklas Luhmann — mỗi note có ID unique, linked với các notes liên quan. |
| **PARA** | Hệ thống tổ chức của Tiago Forte: Projects, Areas, Resources, Archives. |
| **Knowledge Graph** | Mạng lưới các knowledge nodes được kết nối bằng links — càng nhiều connections = càng valuable. |
| **Signal-to-Noise Ratio** | Tỷ lệ thông tin có giá trị trên tổng thông tin thu thập — KPI quan trọng của capture pipeline. |
| **Knowledge Debt** | Thông tin đã capture nhưng chưa được process — càng nhiều = càng nợ. |
| **Review Cycle** | Chu kỳ review định kỳ vault: daily (fleeting notes), weekly (projects), monthly (areas), quarterly (full vault). |
| **Personal Growth Score** | Chỉ số tổng hợp đo lường sự phát triển cá nhân — bao gồm skills, knowledge depth, project outcomes. |
| **Learning Goal** | Mục tiêu học tập cụ thể, có thể đo lường, có deadline — không phải "muốn biết về AI" mà là "hoàn thành fast.ai course trước 2026-06-30". |
| **Knowledge Transfer** | Quá trình chuyển giao tri thức từ cá nhân/vault sang người khác hoặc dự án — output cuối cùng của knowledge pipeline. |
| **Source** | Bất kỳ nguồn thông tin nào: sách, bài báo, podcast, video, conversation, experiment, observation. |
| **Inbox** | Vùng tiếp nhận thông tin chưa được xử lý — cần được cleared trong 24-48h. |
| **Processing** | Bước biến fleeting note thành atomic note hoặc project note — add context, tags, links. |
