## 0. IPO Analysis

| Component | Detail |
|-----------|--------|
| **INPUT** | Deliverable hoặc insights ready for sharing + Target audience |
| **CONTROL** | Transfer quality: recipient must be able to ACT on the knowledge |
| **OUTPUT** | Knowledge transferred: recipient understands + can apply |
| **MECHANISM** | Meeting, documentation, teaching, pair work, written guide |

---

## 1. Knowledge Transfer Modes

| Mode | When | Format | Duration |
|------|------|--------|----------|
| **Show & Tell** | Quick insight sharing | 5-min demo | 5-15 min |
| **Written Guide** | Reusable reference | SOP/tutorial | 1-4h to create |
| **Workshop** | Complex skill transfer | Interactive session | 1-2h |
| **Pair Work** | Tacit knowledge | Working together | Ongoing |
| **Documentation** | Org knowledge | Wiki/handbook | Continuous |

---

## 2. Quy Trình

| # | Hành động | Chi tiết | Output |
|---|----------|----------|--------|
| 1 | Define transfer goal | "Sau buổi này, recipient sẽ có thể làm gì?" | Clear goal |
| 2 | Choose transfer mode | Based on knowledge type and urgency | Mode selected |
| 3 | Prepare transfer material | Simplify gems for recipient context | Transfer material |
| 4 | Execute transfer | Meeting/write/demo | Transfer executed |
| 5 | Verify understanding | Ask recipient to explain back or demonstrate | Verified |
| 6 | Create follow-up resource | Doc/note they can reference later | Reference created |
| 7 | Log transfer | Track: who, what, when, outcome | Transfer log |

---

## 3. Checklist
- [ ] Transfer goal defined (what recipient can DO)?
- [ ] Understanding verified (not just "did you get it?")?
- [ ] Follow-up resource created?
- [ ] Transfer logged?
