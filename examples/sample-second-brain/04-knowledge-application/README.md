# 04 — Knowledge Application Department

**Layer:** Operations (L2) — Primary Activity #4
**Head:** Head of Application
**Team size:** 3

## Department IPO

| Component | Detail |
|-----------|--------|
| **INPUT** | Active projects + Vault (gems, syntheses, relevant notes) |
| **CONTROL** | "Apply or Lose" principle, Project quality standards, Knowledge attribution requirement |
| **OUTPUT** | Knowledge-powered project deliverables, lessons learned, knowledge transfer artifacts |
| **MECHANISM** | Vault search, Claude AI, Project brief template, vibe-ebook-orchestrator, vibe-second-brain |

## KPIs

| KPI | Metric | Target | Frequency |
|-----|--------|--------|-----------|
| Knowledge Utilization Rate | Projects using ≥2 gems / total projects | ≥80% | Monthly |
| Deliverables Created | # knowledge-backed deliverables | ≥3/month | Monthly |
| Transfer Success | % recipients who apply knowledge | ≥70% | Monthly |
| Application Diversity | # unique domains where knowledge applied | ≥2/month | Monthly |
| Return on Knowledge | Impact of applied knowledge (qualitative) | Reviewed quarterly | Quarterly |
